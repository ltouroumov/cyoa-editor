# V1 Format (Viewer)

The V1 format is the production format used by the viewer for compatibility with the MeanDelay iCYOA ecosystem.

## Structure

```json
{
  "$projectId": "optional-unique-id",
  "rows": [...],
  "backpack": [...],
  "pointTypes": [...],
  "styling": {...}
}
```

## Hierarchy

```bash
Project
├── rows[]                 # Main content rows
│   ├── id                 # Unique identifier
│   ├── title              # Row header title
│   ├── titleText          # Row description
│   ├── image              # Header image URL or base64
│   ├── imageIsLink        # true = URL, false = base64
│   ├── objectWidth        # Bootstrap column class (e.g., "col-md-4")
│   ├── requireds[]        # Display conditions
│   ├── allowedChoices     # Max selections limit
│   └── objects[]          # Choices in this row
│       ├── id
│       ├── title
│       ├── text
│       ├── image
│       ├── scores[]       # Point modifications
│       ├── addons[]       # Sub-items
│       └── requireds[]    # Selection conditions
├── backpack[]             # Backpack rows (same structure as rows)
├── pointTypes[]           # Point type definitions
│   ├── id
│   ├── name
│   ├── startingSum
│   ├── beforeText         # Text before value
│   └── afterText          # Text after value
└── styling                # Global style settings
    ├── backgroundColor
    ├── rowTitle, rowText, etc.
    └── objectTitle, objectText, etc.
```

## Width Classes

V1 uses Bootstrap column classes that map to a 60-column grid:

| Class      | Width (of 60)   |
| ---------- | --------------- |
| `col-12`   | 60 (full width) |
| `col-sm-6` | 30 (half)       |
| `col-md-4` | 20 (third)      |
| `col-md-3` | 15 (quarter)    |
| `col-lg-2` | 10 (sixth)      |
| `col-xl-1` | 5 (twelfth)     |

## Condition Terms

Requirements use a nested structure:

```json
{
  "requireds": [
    {
      "type": "id", // "id" or "or"
      "required": true, // true = required, false = incompatible
      "reqId": "choice-id", // Primary ID
      "reqId1": "", // Additional IDs (for type: "id")
      "orRequired": [
        // For type: "or"
        { "req": "choice-a" },
        { "req": "choice-b" }
      ],
      "showRequired": true, // Display requirement text
      "requireds": [] // Nested conditions (AND)
    }
  ]
}
```
